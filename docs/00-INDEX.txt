PRINTED QUICK REFERENCE

ESSENTIAL COMMANDS
===================
Start:          docker compose up -d
Stop:           docker compose down
Logs:           docker compose logs -f core
Restart:        docker compose restart core
Migrate:        docker compose exec core python manage.py migrate
Shell:          docker compose exec core python manage.py shell
DB Shell:       docker compose exec db psql -U clashcode -d clashcode_db
Redis CLI:      docker compose exec redis redis-cli
Test:           docker compose exec core python manage.py test
Health:         curl http://localhost/health/
Status:         docker compose ps

ACCESS POINTS
==============
Frontend:       http://localhost:5173
API:            http://localhost/api/
Admin:          http://localhost/admin/
API Docs:       http://localhost/api/docs/

DOCUMENTATION MAINTENANCE

Updating Documentation:
========================
When you make changes to the system, update relevant docs:

- New feature? → Update 01-PROJECT-OVERVIEW.txt
- API changes? → Update 04-API-REFERENCE.txt
- New command? → Update 05-COMMANDS-REFERENCE.txt
- Security fix? → Update 06-SECURITY-GUIDE.txt
- Deploy changes? → Update 07-DEPLOYMENT-GUIDE.txt
- New issue found? → Update 08-TROUBLESHOOTING.txt

Format:
- Use plain text for maximum compatibility
- Use === for major sections
- Use --- for subsections
- Use ✓ and ✗ for checklists
- Keep lines under 80 characters

Review Schedule:
- Monthly: Review for accuracy
- Quarterly: Major update check
- Annually: Complete rewrite if needed
