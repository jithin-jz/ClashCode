================================================================================

# Stop all services (keeps data)

================================================================================
DATABASE OPERATIONS
================================================================================

# Backup PostgreSQL database
docker compose exec db pg_dump -U clashcode clashcode_db > backup.sql

# Restore from backup
cat backup.sql | docker compose exec -T db psql -U clashcode -d clashcode_db

# Backup with compression
docker compose exec db pg_dump -U clashcode clashcode_db | gzip > backup.sql.gz

# Restore compressed backup
gunzip -c backup.sql.gz | docker compose exec -T db psql -U clashcode -d clashcode_db

# Backup Redis data
docker compose exec redis redis-cli BGSAVE
docker cp clashcode-redis-1:/data/dump.rdb ./redis-backup.rdb

# View database size
docker compose exec db psql -U clashcode -d clashcode_db -c "SELECT pg_size_pretty(pg_database_size('clashcode_db'));"

================================================================================
TROUBLESHOOTING
================================================================================

Problem: Service won't start
Solution:
1. Check logs: docker compose logs core
2. Verify .env files are configured
3. Check dependencies: docker compose ps

Problem: Port already in use
Solution:
1. Find process: netstat -tulpn | grep :80
2. Kill process or change port in .env
3. NGINX_HTTP_PORT=8080

Problem: Database connection failed
Solution:
1. Check DB is running: docker compose ps db
2. Check DB logs: docker compose logs db
3. Restart DB: docker compose restart db

Problem: 502 Bad Gateway
Solution:
1. Check backend services: docker compose ps
2. Check nginx logs: docker compose logs nginx
3. Verify services are healthy
4. Restart nginx: docker compose restart nginx

Problem: High memory usage
Solution:
1. Check usage: docker stats
2. Restart heavy services: docker compose restart core chat
3. Increase limits in docker-compose.yml if needed

Problem: Celery tasks stuck
Solution:
1. Check worker logs: docker compose logs worker
2. Purge stuck tasks: docker compose exec core python manage.py celery purge
3. Restart workers: docker compose restart worker beat

Problem: Code execution failing
Solution:
1. Check executor logs: docker compose logs executor
2. Verify Docker access: docker compose exec executor docker ps
3. Test manually: curl -X POST http://localhost:8011/execute -d '{"code": "print(\"test\")"}'

================================================================================
BACKUP AUTOMATION
================================================================================

Create backup.sh:
#!/bin/bash
BACKUP_DIR="/backups/clashcode"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# Database backup
docker compose exec db pg_dump -U clashcode clashcode_db | gzip > $BACKUP_DIR/db_$TIMESTAMP.sql.gz

# Redis backup
docker compose exec redis redis-cli BGSAVE
docker cp clashcode-redis-1:/data/dump.rdb $BACKUP_DIR/redis_$TIMESTAMP.rdb

# Keep only last 7 days
find $BACKUP_DIR -name "*.gz" -mtime +7 -delete
find $BACKUP_DIR -name "*.rdb" -mtime +7 -delete

echo "Backup completed: $TIMESTAMP"

Add to crontab (daily at 2 AM):
0 2 * * * /path/to/backup.sh
