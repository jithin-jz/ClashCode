SERVER SETUP
================================================================================

1. INITIAL SERVER CONFIGURATION
================================

SSH to server:
ssh user@your-server-ip

Update system:
sudo apt update && sudo apt upgrade -y

Install dependencies:
sudo apt install -y \
    apt-transport-https \
    ca-certificates \
    curl \
    gnupg \
    lsb-release \
    software-properties-common

Install Docker:
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

Add user to docker group:
sudo usermod -aG docker $USER
newgrp docker

Install Docker Compose Plugin:
sudo apt install docker-compose-plugin -y

Verify installation:
docker --version
docker compose version

================================================================================
SSL CERTIFICATE SETUP
================================================================================

2. INSTALL CERTBOT
===================

sudo apt install certbot python3-certbot-nginx -y

3. OBTAIN SSL CERTIFICATE
===========================

sudo certbot certonly --standalone -d your-domain.com -d www.your-domain.com

Follow prompts to complete.

Certificate location:
/etc/letsencrypt/live/your-domain.com/fullchain.pem
/etc/letsencrypt/live/your-domain.com/privkey.pem

Auto-renewal (already configured by certbot):
sudo certbot renew --dry-run

cd /opt

6. GENERATE PRODUCTION KEYS
=============================

Generate SECRET_KEY:
openssl rand -base64 48

Generate JWT keys:
openssl genrsa -out jwt_private.pem 4096
openssl rsa -in jwt_private.pem -pubout -out jwt_public.pem

Convert to base64 for .env:
cat jwt_private.pem | base64 -w 0
cat jwt_public.pem | base64 -w 0

================================================================================
NGINX CONFIGURATION FOR PRODUCTION
================================================================================

8. UPDATE NGINX CONFIG
========================

Edit nginx/nginx.conf for production:

Add SSL configuration:
server {
    listen 443 ssl http2;
    server_name your-domain.com www.your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    
    # ... rest of config
}

Redirect HTTP to HTTPS:
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;
    return 301 https://$server_name$request_uri;
}

Update frontend location:
location / {
    # In production, serve static files or proxy to frontend container
    root /opt/CLASHCODE/frontend/dist;
    try_files $uri $uri/ /index.html;
}

BACKUP SYSTEM
================================================================================

17. AUTOMATED BACKUPS
======================

Create backup script:
/opt/CLASHCODE/scripts/backup.sh

#!/bin/bash
BACKUP_DIR="/backups/clashcode"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# Database backup
docker compose exec db pg_dump -U clashcode clashcode_db | gzip > $BACKUP_DIR/db_$TIMESTAMP.sql.gz

# Redis backup
docker compose exec redis redis-cli BGSAVE
docker cp clashcode-redis-1:/data/dump.rdb $BACKUP_DIR/redis_$TIMESTAMP.rdb

# Keep last 7 days
find $BACKUP_DIR -name "*.gz" -mtime +7 -delete
find $BACKUP_DIR -name "*.rdb" -mtime +7 -delete

echo "Backup completed: $TIMESTAMP" >> $BACKUP_DIR/backup.log

Make executable:
chmod +x /opt/CLASHCODE/scripts/backup.sh

Add to crontab:
crontab -e

0 2 * * * /opt/CLASHCODE/scripts/backup.sh

docker compose up -d --build

================================================================================
TROUBLESHOOTING
================================================================================

Problem: 502 Bad Gateway
Solution:
1. docker compose ps (check services)
2. docker compose logs core
3. docker compose restart nginx
4. Check SSL certificates

Problem: Database connection refused
Solution:
1. docker compose ps db
2. docker compose logs db
3. Check POSTGRES_PASSWORD in .env
4. docker compose restart db

Problem: High memory usage
Solution:
1. docker stats
2. Increase server RAM
3. Optimize container limits
4. Restart memory-heavy services

Problem: SSL certificate expired
Solution:
sudo certbot renew
sudo systemctl reload nginx

Problem: Code execution timeout
Solution:
1. Check executor logs
2. Increase timeout in .env
3. Check resource limits
4. Restart executor
