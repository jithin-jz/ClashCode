ISSUE 1: Service Won't Start
==============================

Symptoms:
- Container shows "Exited" or "Restarting"
- docker compose ps shows unhealthy status

Diagnosis:
docker compose logs core
docker compose ps

Common Causes:
1. Missing environment variables
2. Database not ready
3. Port already in use
4. Invalid configuration
5. Dependency not running

Solutions:

A. Check environment variables:
docker compose exec core env | grep SECRET_KEY

Fix:
Edit .env file and ensure all required variables are set.
See SETUP.txt for required variables.

B. Check dependencies:
docker compose ps db
docker compose ps redis

Fix:
docker compose up -d db redis
Wait for them to be healthy, then:
docker compose up -d core

C. Check for port conflicts:
netstat -tulpn | grep :80

Fix:
Kill the process using the port or change port in .env:
NGINX_HTTP_PORT=8080

D. View detailed logs:
docker compose logs --tail=100 core

Look for error messages and stack traces.

E. Restart service:
docker compose restart core

ISSUE 2: Database Connection Failed
=====================================

Symptoms:
- "Connection refused" error
- "FATAL: password authentication failed"
- Core service keeps restarting

Diagnosis:
docker compose logs db
docker compose logs core | grep database

Solutions:

A. Verify database is running:
docker compose ps db

If not running:
docker compose up -d db

B. Check database logs:
docker compose logs db

Look for:
- Startup errors
- Authentication failures
- Disk space issues

C. Test connection:
docker compose exec core python manage.py dbshell

If it connects, database is working.

D. Verify credentials:
Check services/core/.env:
DB_HOST=db
DB_NAME=clashcode_db
DB_USER=clashcode
DB_PASSWORD=<matches docker-compose.yml>

E. Reset database (DESTRUCTIVE - loses all data):
docker compose stop core worker beat
docker compose down -v db
docker compose up -d db
docker compose exec core python manage.py migrate

ISSUE 3: 502 Bad Gateway (Nginx)
==================================

Symptoms:
- Browser shows "502 Bad Gateway"
- API requests fail

Diagnosis:
docker compose logs nginx
docker compose ps

Solutions:

A. Check backend services:
docker compose ps

All should be "Up" and "healthy"

B. Check nginx logs:
docker compose logs nginx

Look for:
- "connection refused"
- "upstream prematurely closed connection"
- "no live upstreams"

C. Test backend directly:
docker compose exec core curl http://localhost:8000/health/

If this works, backend is fine. Issue is nginx config.

D. Restart nginx:
docker compose restart nginx

E. Check nginx configuration:
docker compose exec nginx nginx -t

Should show "syntax is ok" and "test is successful"

F. Verify services are on correct ports:
docker compose exec core netstat -tlnp

Should show port 8000 listening

ISSUE 4: Code Execution Failing
=================================

Symptoms:
- "Execution failed" error
- Timeout errors
- No output returned

Diagnosis:
docker compose logs executor
Check error message in response

Solutions:

A. Check executor logs:
docker compose logs executor

Look for:
- Docker errors
- Resource limit exceeded
- AST validation failures

B. Verify Docker access:
docker compose exec executor docker ps

Should work without errors.

C. Test execution manually:
curl -X POST http://localhost:8011/execute \
  -H "Content-Type: application/json" \
  -d '{"code": "print(\"test\")", "stdin": ""}'

Expected response:
{"success": true, "stdout": "test\n", ...}

D. Check resource limits:
docker compose exec executor free -m
docker compose exec executor nproc

E. Increase limits if needed:
Edit services/docker-compose.yml:
executor:
  mem_limit: 256m  # Increase from 128m
  cpus: 1.5        # Increase from 1.0

F. Restart executor:
docker compose restart executor

G. Check for blocked imports:
If user code uses: os, sys, subprocess, etc.
It will be blocked by AST validation.

ISSUE 5: WebSocket Connection Issues
======================================

Symptoms:
- Chat not connecting
- "WebSocket disconnected" error
- Messages not sending/receiving

Diagnosis:
Browser DevTools → Console → WebSocket errors
docker compose logs chat

Solutions:

A. Check chat service:
docker compose ps chat

Should be "Up" and "healthy"

B. Check nginx WebSocket config:
cat nginx/nginx.conf

Should include:
map $http_upgrade $connection_upgrade {
    default upgrade;
    "" close;
}

location /ws/ {
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection $connection_upgrade;
}

C. Test WebSocket connection:
Install wscat:
npm install -g wscat

Test:
wscat -c ws://localhost/ws/chat/

Should connect successfully.

D. Check Redis Pub/Sub:
docker compose exec redis redis-cli
PUBSUB CHANNELS

Should show chat channels.

E. Restart chat service:
docker compose restart chat

ISSUE 6: Token Refresh Loop
==============================

Symptoms:
- Infinite login/logout cycle
- Multiple 401 errors
- "Token refresh failed" messages

Diagnosis:
Browser DevTools → Network → Check /auth/refresh/ calls
docker compose logs core | grep refresh

Solutions:

A. Check Redis for blacklisted tokens:
docker compose exec redis redis-cli
KEYS blacklist:*

If many keys, might be causing issues.

B. Clear browser cookies:
1. Open DevTools
2. Application → Cookies
3. Clear all for localhost
4. Reload page

C. Verify JWT configuration:
docker compose exec core python manage.py shell

>>> from django.conf import settings
>>> print(settings.JWT_PRIVATE_KEY[:50])
>>> print(settings.JWT_PUBLIC_KEY[:50])

Should show key data.

D. Check token expiry:
Inspect JWT token at https://jwt.io
Check exp (expiry) timestamp

E. Reset user tokens:
docker compose exec core python manage.py shell

>>> from authentication.models import User
>>> user = User.objects.get(username='coder123')
>>> user.blacklist_all_tokens()

ISSUE 7: High Memory Usage
=============================

Symptoms:
- Server slowdown
- Out of memory errors
- Container restarts

Diagnosis:
docker stats
free -m

Solutions:

A. Check memory usage:
docker stats --no-stream

Identify memory-heavy containers.

B. Restart heavy services:
docker compose restart core chat

C. Increase container limits:
Edit docker-compose.yml:
core:
  mem_limit: 2G
  mem_reservation: 1G

D. Check for memory leaks:
docker compose logs core | grep -i memory

Look for increasing memory usage over time.

E. Optimize database queries:
Slow queries can cause memory issues.

docker compose exec core python manage.py shell

>>> from django.db import connection
>>> print(len(connection.queries))

If large number, queries not being cleared.

F. Increase server RAM:
If consistently high, upgrade server.

ISSUE 8: Celery Tasks Stuck
==============================

Symptoms:
- Tasks not completing
- "Task queued" but not executing
- Background operations failing

Diagnosis:
docker compose logs worker
docker compose logs beat

Solutions:

A. Check worker status:
docker compose ps worker

Should be "Up"

B. Check worker logs:
docker compose logs worker

Look for:
- Connection errors
- Task failures
- Import errors

C. List active tasks:
docker compose exec core python manage.py shell

>>> from celery.app.control import Control
>>> from project.celery import app
>>> inspect = app.control.inspect()
>>> print(inspect.active())

D. Purge stuck tasks:
docker compose exec core python manage.py celery -A project purge

WARNING: This deletes all queued tasks!

E. Restart workers:
docker compose restart worker beat

F. Check Redis (message broker):
docker compose exec redis redis-cli
KEYS celery*

If queue is huge, might need to purge.

G. Scale workers:
docker compose up -d --scale worker=3

ISSUE 9: Email Not Sending
=============================

Symptoms:
- OTP emails not received
- Notification emails failing

Diagnosis:
docker compose logs core | grep email
Check SMTP credentials

Solutions:

A. Verify SMTP settings:
Check services/core/.env:
EMAIL_BACKEND=django.core.mail.backends.smtp.EmailBackend
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USE_TLS=True
EMAIL_HOST_USER=your@gmail.com
EMAIL_HOST_PASSWORD=your-app-password

B. Test email:
docker compose exec core python manage.py shell

>>> from django.core.mail import send_mail
>>> send_mail(
...     'Test',
...     'Test message',
...     'from@example.com',
...     ['to@example.com'],
...     fail_silently=False,
... )

Should return 1 (sent).

C. Check SMTP provider:
- Gmail: Use App Password, not account password
- AWS SES: Verify domain, check limits
- Other: Verify credentials

D. Check firewall:
Ensure port 587 is not blocked.

E. Use console backend for testing:
EMAIL_BACKEND=django.core.mail.backends.console.EmailBackend

Emails will print to console.

ISSUE 12: SSL Certificate Errors
===================================

Symptoms:
- Browser security warnings
- "Certificate expired"
- "Certificate not trusted"

Diagnosis:
curl -v https://your-domain.com
Check certificate dates

Solutions:

A. Check certificate expiry:
openssl x509 -enddate -noout -in /etc/letsencrypt/live/your-domain.com/cert.pem

B. Renew certificate:
sudo certbot renew
sudo systemctl reload nginx

C. Auto-renewal setup:
sudo certbot renew --dry-run

Should work automatically via cron.

D. Check nginx SSL config:
ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

E. Test SSL:
https://www.ssllabs.com/ssltest/

ENABLE DEBUG MODE (Development Only)
======================================

1. Edit services/core/.env:
DEBUG=True

2. Restart core:
docker compose restart core

3. Access debug toolbar:
http://localhost/api/__debug__/

WARNING: Never enable DEBUG=True in production!

ENABLE VERBOSE LOGGING
========================

Edit services/core/.env:
LOG_LEVEL=DEBUG

Restart:
docker compose restart core

ADVANCED DEBUGGING
================================================================================

PYTHON DEBUGGER (PDB)
=======================

Add breakpoint in code:
import pdb; pdb.set_trace()

Attach to running container:
docker attach clashcode-core-1

Use pdb commands:
n (next)
c (continue)
p variable (print)
q (quit)

DATABASE QUERY DEBUGGING
==========================

Enable query logging:
LOGGING = {
    'loggers': {
        'django.db.backends': {
            'level': 'DEBUG',
        },
    }
}

VIEW DOCKER NETWORK TRAFFIC
=============================

Install tcpdump:
docker compose exec core apt update
docker compose exec core apt install tcpdump

Capture traffic:
docker compose exec core tcpdump -i eth0 -n
